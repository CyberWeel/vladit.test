# Краткое описание решения

Сразу должен предупредить: задача не доведена до конца. В моём письме должен быть архив с 4 скриншотами того, что у меня получилось выполнить.

- Я установил "Битрикс: Стандарт" локально. Запустил Webpack и Git.
- После анализа архитектуры модуля решил вносить правки именно в файле: https://github.com/CyberWeel/vladit.test/blob/master/bitrix/modules/vit.doctor/lib/api/record.php
- Из-за того, что точно специфика выполнения задачи не упомянута, я решил не грузить таблицу "vd_schedule" ежедневным скриптом с Cron, генерирующим расписание для каждого врача. Я отключил получение данных о расписании врача из базы данных и сделал более по-простому - в админке указывается время для дней недели, и только эти дни недели будут доступны в указанный промежуток времени со слотами по 20 минут.
- Добавил в инфоблок "Врачи" 7 полей - "Понедельник", "Вторник", ..., "Воскресенье". Их можно заполнять подобным образом (это строковые поля): "11:00 - 19:00". В таком случае код разберёт подобную запись на время начала и конца рабочего дня, а если время вовсе для дня недели не указано - то этот день считается выходным (на одном из скриншотов видна подсказка). Также, код был протестирован и варианты полей "17:00", "99:99", "абвгд" корректно воспринимались как выходные дни.

Не доделана задача в том плане, что блок частично сломался, левая его часть не выводится, потому что массив $arSchedule не заполняется как нужно, но я решил прекратить выполнение задачи, т.к. вышел за рамки 3 часов, отведённых на задачу и всё равно решил продемонстрировать вам мой ход мыслей.

В любом случае буду рад услышать обратную связь!
